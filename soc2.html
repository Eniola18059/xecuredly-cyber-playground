Understanding Web Servers & Architecture

What is a Web Server?

A web server is a software and hardware system that uses HTTP and other protocols to respond to client requests made over the World Wide Web. It stores, processes, and delivers web pages to users.

How Web Servers Work:

  • Client (browser) sends a request for a webpage (e.g., www.xecuredly.com)
  • DNS resolves the domain to an IP address
  • Web server receives the request and returns the webpage files (HTML, CSS, JS)
  • The browser renders the webpage for the user

Web Architecture Components:

  • Client: Browser/device sending the request
  • Web Server: Receives request, serves web pages
  • Application Server: Handles logic, authentication, etc.
  • Database Server: Stores and retrieves data

Common Web Technologies:

  • HTTP / HTTPS
  • TCP/IP Protocol
  • DNS (Domain Name System)
  • Load Balancers

Web Server Software:

  • Apache HTTP Server
  • Nginx
  • Microsoft IIS

Common Web Server Vulnerabilities:

  • Directory Traversal Attacks
  • Misconfigured Permissions
  • Unpatched Server Software
  • Information Disclosure
